Getting Started

Set up your Syft NSAI Router in minutes and start building intelligent AI services

Getting Started with Syft NSAI Router

1. Install SyftBox

Download SyftBox for your platform or use the CLI installer.

Get SyftBox →
CLI (macOS/Linux):
curl -fsSL https://syftbox.net/install.sh | sh

2. Add Your App via GitHub URL

In SyftUI, click Add App and paste your app’s GitHub URL.
Or use the CLI to add your app.

SyftBox will:
  • Clone your app from GitHub
  • Recognize run.sh as the entry point
  • Automatically start your app for you (no need to run run.sh manually)
  • Show logs and dashboard location in SyftUI or CLI

Manual Setup (Advanced/Optional)

You can still run your app manually for development or advanced use cases. See the README for details.

Installation & Setup

1

Clone the Repository

Start by cloning the Syft NSAI Router repository to your local machine.

Terminal 
# Clone the repository
git clone <repository-url>

# Navigate to the project directory
cd syft-llm-router

# Verify the structure
ls -la

Note: Make sure you have Git installed and configured with your credentials before cloning.

2

SyftBox Auto-Start

If SyftBox is running, it will automatically initiate your app when you add it. You do not need to run run.sh manually. You can view logs and find the dashboard location directly in SyftUI or via the CLI.

Tip: Use SyftUI to monitor your app's status, view logs, and access the dashboard URL.

4

Verify Installation

To confirm your app is running, open SyftUI or use the CLI. You will see your app listed, and the dashboard link will be displayed in the UI or logs. Click the dashboard link to access your running router.

Tip: The dashboard link is shown in the SyftUI app list and in the logs after startup. No need to manually enter a URL.

Configuration

Environment Variables

SYFTBOX_ASSIGNED_PORT

Custom port for the server (default: 8080)

SYFTBOX_CONFIG_PATH

Path to SyftBox configuration file

DATABASE_PATH

Custom database location (default: data/)

# Example .env file
SYFTBOX_ASSIGNED_PORT=9000
DATABASE_PATH=./custom_data
DEBUG=true

Custom Port Setup

If port 8080 is already in use, you can specify a different port:

# Set custom port before running
export SYFTBOX_ASSIGNED_PORT=9000
./run.sh

# Or run with inline environment variable
SYFTBOX_ASSIGNED_PORT=9000 ./run.sh

The system will automatically detect if the default port is in use and suggest alternatives.
