Federated RAG

Federated Retrieval-Augmented Generation (RAG) enables the NSAI Protocol to search and combine knowledge from multiple, independently governed data owners‚Äîeach under their own terms and permissions. Unlike traditional RAG, which relies on a single, centralized data source, Federated RAG orchestrates queries across a distributed mesh of datasites, aggregating results in real time while preserving privacy and attribution.

In the SyftBox context, each data owner exposes a search endpoint (following OpenAPI spec) that the protocol can route queries to. The aggregator coordinates these queries, respects each owner's access policies, and merges the results for the user. This approach unlocks access to a vastly larger and more diverse set of knowledge, while ensuring that data never leaves its source and that every contribution is properly attributed and compensated.

NSAI Overview

What is NSAI?

NSAI (Network Sourced AI) is an open-source protocol that combines private and public models into powerful ensembles while keeping users in control. It addresses the problem that most of the world's intelligence is locked away in private datasets, proprietary models, and local environments, where centralization isn't feasible or scalable.

Key Participants

  • Private Model Providers - Serve private data or models while maintaining privacy for monetization
  • Model Users - Companies and developers who consume these models for superior results
  • Model Hosts - Provide infrastructure to serve models for compensation

SyftBox Overview

Privacy-Preserving Protocol

SyftBox is an open-source protocol that enables privacy-preserving computations across distributed datasets without requiring data centralization. It allows developers and organizations to run analyses on data that remains securely with its original owners, never being copied or moved.

Network Architecture

The platform operates by creating a network of "datasites" similar to how the internet connects websites. Each datasite is essentially a folder on someone's computer that they maintain full control over, with Unix-like permissions governing access.

Key Advantages

  • Simplified development across different programming languages
  • Secure collaboration while maintaining privacy
  • Enhanced trust through auditable frameworks
  • True privacy by design with complete control

Model Ensembling & MoE

Model Ensembling & Mixture of Experts (MoE) enables the NSAI Protocol to combine insights from multiple models, each with their own terms and weights. When a query is made, each model responds independently. The protocol aggregates these responses, applying weights and logic to produce a superior, consensus-based answer. This approach increases accuracy, robustness, and trust, as users can see which models contributed and how their answers were combined.

Permissioned AI Interfaces

Permissioned AI Interfaces ensure that every query is subject to explicit approval or denial by data owners. When a user submits a query, each owner can approve or deny access in real time, just like a browser requesting permission to access a website. This mechanism enforces privacy, compliance, and trust, and is a core part of attribution-based control in NSAI.

Unified Access

Unified Access means every data or model owner can expose standardized endpoints (chat/search) using OpenAPI, making it easy for users and systems to discover and interact with resources across the network. This enables seamless integration, interoperability, and automation, while maintaining owner control and attribution.

Interoperable

Interoperable means the protocol and its endpoints are reachable from anywhere‚ÄîHTTP, syftRPC, local, cloud, or third-party. Data and models can be served from any environment, enabling a truly global, flexible, and future-proof AI network.
