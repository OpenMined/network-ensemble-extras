1

Create Your Router

Start by creating your first AI router through the SyftBox dashboard. This process generates a complete project structure with all necessary files and configurations. Your router will handle both chat and search requests, making your AI services accessible to the federated network.

Router Creation Process

  1. Navigate to your SyftBox dashboard
  2. Click "Create New Router"
  3. Choose your router profile: Provider (to offer services)
  4. Select router type: Default (recommended) or Custom
  5. Choose services: Chat, Search, or both
  6. Configure basic settings and click "Create"
✨ Example: Router Creation Dialog

Fill in your project name and follow the 4-step wizard to create your router.

Router Types Explained:

  • Default Router: Pre-configured with Ollama for chat and local RAG for search. Perfect for beginners.
  • Custom Router: Empty template requiring manual implementation of endpoints and logic.

Generated Files:

  • chat_router.py - Handles chat/completion requests
  • service_router.py - Manages search/retrieval requests
  • run.sh - Startup script for SyftBox
  • requirements.txt - Python dependencies
🎯 Result: Router Overview Dashboard

Your router is created in "Draft" status with both Chat and Search endpoints available. Notice the "Publish Router" button to make it live.

2

Configure Data Sources

Transform your local data into searchable knowledge by setting up the indexing system. This crucial step enables your router's search capabilities and creates a powerful RAG (Retrieval-Augmented Generation) system that enhances chat responses with your specific data.

Setup Process

  1. Open your SyftBox dashboard and locate your router app
  2. Click "Open Website" to access the router interface
  3. Navigate to the "Data Sources" or "Indexing" section
  4. Select folders, files, or databases you want to make searchable
  5. Configure chunking settings (default: 1000 characters with 200 overlap)
  6. Click "Start Indexing" and wait for completion

Data Indexing Process:

  1. Navigate to your router app in SyftUI dashboard
  2. Click the "Open Website" or dashboard link
  3. Look for "Data Indexing" or "RAG Configuration" section
  4. Select local folders or files to index
  5. Configure chunking settings (optional)
  6. Start the indexing process

Supported Data Sources:

  • Local Files: .txt, .md, .rst, .html, .pdf, .docx, .pptx
  • Code Repositories: .py, .js, .java, .cpp, .go, .rs, etc.
  • Structured Data: .csv, .json, .xml, .yaml
  • Data Source Platforms: WordPress, Automattic, SilverChair, Confluence

Universal Data Integration

NSAI Protocol supports data from any platform or service. Whether your content lives in cloud services, content management systems, or proprietary databases, you can make it searchable and available through your AI router while maintaining complete data ownership.

  • Content Platforms: WordPress, Drupal, Ghost, Medium
  • Cloud Storage: Google Drive, Dropbox, OneDrive, S3
  • Publishing Systems: SilverChair, HighWire, Scholastica
  • Collaboration Tools: Slack, Microsoft Teams, Discord
  • Knowledge Bases: Notion, Confluence, GitBook, Bookstack
  • Custom APIs: Any REST or GraphQL endpoint

Publish to the Network

Make your router available to the global federated network by publishing it with proper metadata and pricing. This step transforms your local router into a discoverable service that can generate revenue while maintaining data ownership and privacy.

Publishing Checklist

📝 Example: Publish Router Form

Complete the 6-step publishing wizard with router name, summary, description, tags, and pricing. The example shows $0 for chat and $0.30 for search requests.

Publishing Steps:

  1. Fill in router metadata (name, description, tags)
  2. Set pricing model (per request, subscription, etc.)
  3. Configure access permissions and rate limits
  4. Add usage examples and documentation
  5. Review and confirm publication
4

Test, Monitor & Optimize

Ensure your router performs optimally by conducting thorough testing, setting up monitoring, and optimizing based on usage patterns. This ongoing process helps maintain service quality and maximizes your revenue potential in the federated network.

Testing Protocol

  1. Chat Testing: Use the web interface to test conversational responses
  2. Search Testing: Query your indexed data to verify retrieval accuracy
  3. API Testing: Test endpoints programmatically with sample requests
  4. Load Testing: Verify performance under multiple concurrent requests
  5. Error Handling: Test edge cases and invalid inputs

Testing Methods:

  • Web Interface: Built-in chat UI for direct testing
  • cURL: Command-line API testing
  • Python SDK: Programmatic integration

Example API Calls:

# Chat endpoint
curl -X POST "http://localhost:8080/chat" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

# Search endpoint
curl -X POST "http://localhost:8080/search" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "your search query",
    "top_k": 5
  }'

Monitoring Dashboard:

  • Usage Analytics: Track request volume and performance
  • Revenue Tracking: Monitor earnings from router usage
  • Error Logs: View and debug issues
  • Health Status: Real-time router status monitoring
